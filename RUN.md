# Драконоборец — инструкция для крон-сессии Claude Code

Эта сессия запускается по cron (или вручную) и готовит ежедневный отчёт за вчерашний день для Валерия (CMO Zerocoder).

## Что делать

1. **Собрать метрики.** Запусти:
   ```bash
   python metrics.py --out out/latest.json
   ```
   Это пишет JSON с тремя блоками: `adult_webinars`, `leadmagnets`, `kids`.
   Запрос идёт ~30-60 сек (NOT EXISTS по getcourseRegistrations 2М+).

2. **Прочитай `out/latest.json`.** Разберись в структуре:
   - `adult_webinars.launches[]` — каждый элемент `{launch, new_regs, by_channel{канал: N}}`
   - `leadmagnets.launches[]` — то же самое для гайдов/квизов
   - `kids.channels_yesterday` vs `kids.channels_prev` — для сравнения

3. **Прочитай `STYLE.md`** в этой папке — гайд по тону.

4. **Напиши отчёт** в файл `out/latest.txt`. Скелет см. ниже. Живым текстом, «панк в пиджаке»:
   - Называй чемпионов и проигравших — кто разорвал, кто подсдулся
   - Сравнения с позавчерашним (для детских есть готовые числа, для взрослых — смотри глазами на порядок)
   - Коротко, без лишних прилагательных. Лучше цифра + одна меткая фраза, чем абзац воды
   - Лончи называй как есть (`chatbot-intensive-35`, не расшифровывай)
   - Эмодзи — умеренно, как маркеры секций (не в каждой строке)

5. **Отправь в ТГ**:
   ```bash
   python send.py out/latest.txt
   ```

6. **Заверши сессию.** Ничего лишнего не делай.

```

<i>{дата: "20 апр · понедельник"}</i>

━━━━━━━━━━━━━━━━━━━━
🎪 <b>ВЗРОСЛОЕ</b>  ·  {total_new} уников на вебинары
━━━━━━━━━━━━━━━━━━━━
<b>{лонч}</b> — {N}   <i>{канал N · канал N · ...}</i>
<b>{лонч}</b> — {N}   <i>{...}</i>
...

<i>Ещё {X} лончей по {min-max} рег</i>

<blockquote>{один нейтральный комментарий 1-2 предложения}</blockquote>

━━━━━━━━━━━━━━━━━━━━
🧲 <b>ЛИДМАГНИТЫ</b>  ·  {total_new} уников
━━━━━━━━━━━━━━━━━━━━
{так же как взрослое}

━━━━━━━━━━━━━━━━━━━━
🧒 <b>ДЕТСКОЕ</b>  ·  {total_yesterday} заявок  ·  <i>позавчера {total_prev}</i>
━━━━━━━━━━━━━━━━━━━━
<i>(не уники — все заявки за день)</i>
<b>{канал}</b> — {N}   <i>({+/−N})</i>
<b>{канал}</b> — {N}   <i>({+/−N})</i>
...
<blockquote>{один нейтральный комментарий}</blockquote>
```

## Правила вёрстки (Telegram HTML)

- Поддерживаются только: `<b>`, `<i>`, `<u>`, `<s>`, `<code>`, `<pre>`, `<a>`, `<blockquote>`, `<tg-spoiler>`. Никаких `<br>`, `<p>`, `<li>`, таблиц.
- Перевод строки — буквальный `\n`.
- Название лонча / канала жирным, цифра после тире, разбивка в курсиве. Разделитель в разбивке — ` · ` (точка с пробелами).
- Блоки отбивать горизонтальными `━`-линиями (ровно 20 символов).
- Общий комментарий блока — в `<blockquote>` (ТГ рисует вертикальную полоску слева — визуально отделяет от таблицы цифр).
- **Детская таблица — без `<pre>`** (копируется в ТГ кривым моно-текстом). Формат тот же, что у взрослых/лидмагнитов: `<b>канал</b> — N   <i>(+/−N)</i>`.
- Заголовки блоков — с разделителем ` · ` (не двоеточием), эмодзи в начале одинаковые: 🎪 взрослые, 🧲 лидмагниты, 🧒 детское.

## Жёсткие правила

- **По одной строке на лонч/канал.** Только цифра и разбивка в скобках. НИКАКИХ комментариев к отдельной строке.
- **Один общий комментарий на блок** в последней строке после тире — нейтральный, фактический, 1-2 предложения.
- **НЕ упоминай нулевые каналы/лончи.** Если вчера его нет — нет и в отчёте. Никаких «молчит / тишина / нули».
- **НЕ используй образные глаголы** («разорвал», «подсдулся», «вытянул», «умер»). Стиль — сводка, не тост.
- **НЕ придумывай причины** («падение из-за праздников»). Только то, что видишь в данных.
- **Длина** — ≈ 1500 знаков. Хвост лончей схлопывай одной строкой.
- Детское сравнение — через `channels_prev` в JSON. Взрослое и лидмагниты без сравнения (только сегодняшние цифры).

Подробности по тону и примеры — см. `STYLE.md`.

## Путь к БД / кредам

- Все креды в `.env` в корне репо: `TG_BOT_TOKEN`, `TG_CHAT_ID`, `DB_HOST/PORT/USER/PASSWORD/NAME`
- Модули `shared_local/db.py` и `shared_local/filters.py` — внутри репо, не зависят от внешних путей
- Если это первый прогон в окружении — выполни `pip install -r requirements.txt`

## Что НЕ делать

- Не правь `metrics.py`, `send.py` без задачи — они сейчас рабочие
- Не шли в ТГ сырой JSON и не шли черновики. Один финальный отчёт.
- Не пиши «подготовил отчёт» в ТГ — там должен быть только сам отчёт
